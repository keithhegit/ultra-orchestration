Agent-Skills Logic / 技能协作逻辑

The skill stack is role-based. One primary orchestrator skill coordinates specialized sibling skills by phase.

Phase Primary Skill(s) Responsibility
Intake clarify-and-intake Normalize ambiguous requests into actionable intake artifacts.
Plan decision-complete-planner, spec-review Produce dependency-aware plans and pre-implementation spec checks.
Dispatch dispatch-and-track Assign work packages, enforce owned paths, track execution state.
Execute ultra-orchestrator + worker role Implement bounded slices with explicit constraints.
Review code-review, spec-review Run engineering and spec consistency gates with evidence.
QA qa-verify Validate behavior and regressions against acceptance checks.
Deliver deliver-and-retro Assemble final deliverable and progress reporting artifacts.
Retro deliver-and-retro Capture process friction and workflow improvements.
Cross-cutting Safety risk-vetter, safety-guard, autoplan Risk classification, safety constraints, and accelerated planning mode.
Bridge Layer openspec-ultra-bridge Synchronize OpenSpec artifacts with Ultra execution artifacts and slice ledger updates.

Single Authority

Runtime authority is intentionally minimal: skills/ultra-orchestrator + skills/openspec-ultra-bridge.

Slice Discipline

Implement and verify in slices, not milestone-sized coding waves. Advance status only with evidence.

Bridge Synchronization

For each change: keep proposal.md, tasks.md, and ultra-bridge.md synchronized.

Backstory / 研发演进说明

Ultra Orchestration started as a self-designed execution methodology focused on deterministic orchestration, strict phase control, and evidence-based delivery.

Why iterative evolution? In real coding-agent production runs, we repeatedly observed context collapse, hidden regressions, write-conflict risks, and weak traceability when workflows were driven by long, monolithic prompts alone.

This led to a practical design choice inherited from the original CUO whitepaper: before pursuing full autonomous multi-agent scale, first build a structured, role-isolated, rollback-capable, auditable orchestration foundation.

What is the spec-coding trend? Modern AI engineering is moving from prompt-only execution toward spec-first development: durable specs, explicit change ledgers, bounded slices, and evidence-gated status advancement. This improves team handoff quality and release confidence.

As this trend became more influential, Ultra integrated OpenSpec to strengthen long-lived specification assets and change-level governance.

Today, the fusion model is explicit: OpenSpec governs specification truth; Ultra governs execution truth.

The original codex_ultra_orchestration material remains historical R&D context, preserved as backstory rather than executable authority.

Quick Start (EN)

  • Pick or open a Program.
  • Break into Milestone -> Change -> Slice.
  • Run the 8 execution phases in order.
  • Bridge-sync OpenSpec and Ultra artifacts after each slice.
  • Report evidence before advancing status.
