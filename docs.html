English

Ultra Orchestration started as an execution-centric orchestration framework focused on deterministic delivery phases, quality gates, and review loops. As the project matured, OpenSpec was integrated to provide a long-lived specification plane.

The final model is dual-plane:

  • OpenSpec plane: stable specification and status ledger
  • Ultra plane: execution control, QA evidence, and delivery governance

This repository keeps the runnable authority minimal and explicit, while preserving historical context here on this page.

中文

Ultra Orchestration 最初是一套“执行编排优先”的工程方法，强调阶段化交付、质量门禁与审查回环。 随着实践推进，后续引入 OpenSpec，形成了长期规格平面。

最终形成双平面模型：

  • OpenSpec 平面：稳定规格与状态账本
  • Ultra 平面：执行控制、QA 证据与交付治理

本仓库将“可执行权威”保持在最小集合，历史研发叙事则保留在本页面。
