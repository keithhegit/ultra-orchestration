# Security Policy

## Supported Scope

This repository contains skill definitions, references, and process documentation.

## Reporting a Vulnerability

If you discover a security issue (for example: leaked credentials, unsafe scripts, supply-chain concerns), please report responsibly:

1. Do **not** open a public issue with exploit details.
2. Contact the maintainer privately via GitHub security reporting (preferred).
3. Provide:
   - affected file/path
   - impact summary
   - reproduction steps (if applicable)
   - suggested mitigation

## Response Expectations

- Acknowledgement target: within 7 days
- Initial triage target: within 14 days

## Disclosure

Please allow maintainers time to investigate and patch before public disclosure.
