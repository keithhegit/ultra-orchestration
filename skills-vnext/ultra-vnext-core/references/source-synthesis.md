# Source Synthesis

This vNext suite deliberately combines several source patterns.

## From Superpowers

- design before implementation
- one-question-at-a-time clarification
- explicit hard gates
- richer skill-local workflow guidance instead of thin one-line prompts

## From OpenSpec

- proposal, design, tasks, and archive as durable spec assets
- change-based execution units
- long-lived program memory outside chat history

## From gstack

- role discipline
- phase-based work style
- practical safety posture around isolated ownership and integration

## From Karpathy-Guidelines

- think before coding
- simplicity first
- surgical changes
- goal-driven execution with verifiable success criteria

Source: [forrestchang/andrej-karpathy-skills](https://github.com/forrestchang/andrej-karpathy-skills)

## From OMX

- stronger runtime awareness
- durable run folders and execution traces
- lifecycle consciousness from clarification to completion
- keep the base engine, add orchestration value around it

Source: [Yeachan-Heo/oh-my-codex](https://github.com/Yeachan-Heo/oh-my-codex)

## Ultra's Final Position

Ultra remains the control plane.
OpenSpec becomes the high-quality specification front end and long-lived spec
back end.
Superpowers provides the strongest prompt and workflow discipline upgrade.
