# Delivery Checklist

## Before Final Delivery

Confirm:

- accepted implementation or artifact result exists
- review decision is explicit
- QA decision is explicit
- risk decisions are recorded in `vetter_report`
- final output names residual risk
- follow-up items are separated from completed work

## Good Delivery Evidence

- test command and result
- lint or build result
- generated artifact path
- exact files changed
- review findings and resolution
- QA scenarios exercised

## Bad Delivery Evidence

- "looks good" without source
- conclusions based only on intent
- unstated assumptions
- claiming dynamic verification when only static reasoning happened
