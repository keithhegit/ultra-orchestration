# QA Modes

## Static QA

Use when the environment does not allow execution.

Inspect:

- code paths
- diff shape
- test coverage intent
- likely user-visible impact

## Dynamic QA

Use when tests, scripts, or commands can be run safely.

Capture:

- command or test invoked
- important output
- pass or fail
- residual uncertainty

## Minimum Scenario Set

- happy path
- error path
- regression path
- user-visible path
