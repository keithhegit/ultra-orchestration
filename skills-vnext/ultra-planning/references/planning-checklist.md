# Planning Checklist

Before declaring a plan complete, verify:

- the task can be expressed without relying on chat memory
- every work package has explicit owned paths
- parallel candidates have disjoint write scopes
- acceptance checks are testable or reviewable
- retry assumptions are explicit
- non-goals and boundary lines are named
- reviewer and QA stages have concrete inspection targets
- assumptions and blockers are separated
- re-plan triggers are explicit

If any line above is still fuzzy, the plan is not ready.
