# Contributing / 贡献指南

Thank you for your interest in contributing to this repository.

## Scope

This repository hosts reusable orchestration skills and references.

Please keep contributions aligned to:

- OpenSpec + Ultra methodology
- skill quality and clarity
- reproducible workflow guidance

## Contribution Process

1. Fork and create a feature branch.
2. Keep changes focused (one logical update per PR).
3. Update relevant docs together with skill changes.
4. Submit a PR with:
   - summary
   - motivation
   - changed skills/files
   - verification notes

## Skill Change Requirements

When changing a skill:

- keep `SKILL.md` concise and explicit
- update references if behavior changes
- preserve canonical phase order where applicable
- avoid introducing conflicting status vocabulary

## PR Checklist

- [ ] Change scope is clear and minimal
- [ ] README/docs updated if needed
- [ ] No secrets or private data included
- [ ] Methodology consistency preserved
- [ ] Bilingual impact considered (if user-facing text changed)

## Code of Conduct

By participating, you agree to follow [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).
